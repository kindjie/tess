# Second Full Audit — 2026-07-09

Scope: the entire tess repository at the then-identical
`feat/diagnostics-core` and `main` trees. Method: six parallel subsystem
auditors (path/caches, concurrency,
storage/topology/core, diagnostics, tests/benches, build/CI/tooling/docs), an
independent codex cross-audit, and a live ASan/UBSan build + full test run.
All high findings and contested claims were re-verified by hand against
source before inclusion.

**Verdict: healthy.** The 2026-07-06 remediation held — 517/517 tests pass
under ASan/UBSan, CI pinning/secrets/hygiene are clean, the core algorithms
(A* cores, incremental topology, sparse residency, UInt128, trace rings)
survived adversarial re-review. The 3 high findings are all *contract-boundary*
bugs in recently added caching/executor layers, not algorithm bugs.

Severity legend: high = wrong results or deadlock reachable through the
public API as documented; medium = wrong results under plausible misuse,
silent gate/coverage gaps, or doc-vs-behavior contradictions; low =
pathological inputs, hygiene, or hardening.

## High

- **H1. Failure-status route products replay stale NoPath forever.**
  `include/tess/path/path.h:1096-1110` (also portal variant
  `include/tess/path/portal_route.h:181-187`).
  `build_weighted_route_product` records `result.status` unconditionally but
  derives `dependencies_` solely from `product.path_`, which is empty on
  NoPath/InvalidStart. `ChunkVersionDependencies::is_valid` then passes
  vacuously, so `weighted_route_product_path` replays the cached NoPath after
  *any* world edit — including one that opens the route — while
  `weighted_astar_path` returns Found. Contradicts the library's own
  "never mistake 'not searched' for 'no route exists'" doctrine
  (path.h:27-31). Fix direction: failure products must carry a dependency
  that can invalidate (e.g. a world fingerprint / all-chunk version sum), or
  `is_valid` must return false for empty dependency sets.
  *Sources: path auditor + codex, hand-verified.*

- **H2. DistanceFieldProduct misses dependencies on unreached chunks.**
  `include/tess/path/field_product_cache.h` (dependency capture from
  `scratch.touched_` only; blocked tiles are never touched).
  A fully sealed chunk contributes no dependency, so an edit that opens it
  bumps only that chunk's version and `is_valid` stays true. Via
  `process_unit_cached` with `use_unit_field_product_cache=true`, the runtime
  serves the stale product indefinitely (`path_runtime.h:416-431` never
  re-lookups after a hit; `prepare_process` invalidates only the route
  cache) → permanent NoPath for starts in the newly opened chunk. Existing
  test `UnitFieldProductCacheRejectsStaleProductsAfterEdit` covers only
  reached-chunk edits. Same fix family as H1.
  *Sources: path auditor + codex, hand-verified.*

- **H3. `WorkerPoolPhaseExecutor::for_each_operation` has no dispatch
  guard.** `include/tess/ops/phase_executor.h:234-315`.
  Nested dispatch (enqueue-from-callback) deterministically deadlocks: the
  calling worker waits on `done_cv_` for `active_workers_ == 0` but is itself
  an active worker, and the nested call has already reset
  `next_offset_`/`finished_operations_` and overwritten
  `job_context_/job_first_/job_count_`, so other workers re-claim offsets and
  re-execute job A concurrently with job B (data race on user state and
  `results_`). Concurrent dispatch from two external threads corrupts the
  same state without needing the deadlock. The header documents only
  "Callbacks must not throw". Fix direction: document the single-dispatch
  contract and add a debug-build guard (`TESS_ASSERT` on a dispatch flag) at
  minimum. The protocol was verified sound for the intended single-caller
  use (no lost wakeups, correct memory ordering).
  *Sources: concurrency auditor + codex (independent convergence),
  hand-verified.*

## Medium — library correctness / API contracts

- **M1.** `include/tess/ops/queued.h:1067-1083` +
  `include/tess/sim/scheduler.h:46-75` — `execute_plan` aborts mid-plan on
  the first `PolicyMismatch` with earlier ops' world writes already applied,
  and `tick_scheduler_core` skips `mark_pathing_dirty` because
  `stats.executed_ops == false`. Mixed-policy `FrameOps` passes planning →
  world mutated, path caches stay stale → agents path through fresh walls.
  Silent in release.
- **M2.** `include/tess/ops/phase_executor.h:226-231` vs `:307` —
  `reserve_operations` resizes `results_` under `mutex_`, but workers write
  `results_[offset]` and the dispatcher scans it without the mutex; calling
  `reserve_operations` during an active dispatch is a use-after-realloc.
  Same undocumented-contract family as H3.
- **M3.** `include/tess/path/route_cache.h:360` — public
  `cached_astar_path()` never calls `invalidate_if_world_changed()`; only the
  runtime does (`path_runtime.h:302`). Direct users of the documented public
  cache API get stale routes after `mark_dirty` unless they remember the
  invalidation call themselves. Document or auto-check on lookup.
- **M4.** `include/tess/path/route_cache.h:179-188` — `store()` invalidates
  when caps would be exceeded, then inserts unconditionally, so a single
  route larger than `max_path_nodes_` is stored anyway, violating the bound
  (bounded by one route, re-invalidated on next store — but the documented
  cap is not enforced).
- **M5.** `include/tess/path/portal_route.h:99-167` —
  `build_weighted_chunk_portal_route_product` explores only monotone chunk
  staircases and stitches only the best-scoring candidate, so U-shaped
  topologies (or a sealed best seam tile) yield a definitive-looking
  `PathStatus::NoPath` for solvable requests. If this is a deliberate
  heuristic tier, the contract must say so (contrast `Indeterminate`'s
  careful contract).
- **M6.** `include/tess/path/route_cache.h:308-350` +
  `field_product_cache.h:273-308` — world-identity aliasing: fingerprints are
  chunk-version vectors only, so two same-shape worlds with equal version
  vectors (e.g. fixtures that write fields without `mark_dirty`) alias, and a
  runtime/scratch reused across worlds serves wrong-world routes/fields. The
  sparse residency path defends against exactly this (path.h:566-572); dense
  does not.
- **M7.** `include/tess/topology/topology.h:355-372` — `region_index`, the
  documented checked accessor, is bypassed by unsigned wraparound:
  `ChunkKey{UINT64_MAX}` (the sentinel `region_of` itself returns) makes
  `ref.chunk.value + 1` wrap to 0 and pass the guard → OOB read. Secondary
  uint32 wrap on `region_offsets_[i] + ref.region.value - 1` can return a
  wrong-but-valid index.
- **M8.** `include/tess/topology/topology.h:863,1126` — the documented
  "graph not built for this world shape → full rebuild" fallback compares
  only chunk *count*, so an equal-count different-grid shape silently
  incremental-patches on the wrong chunk adjacency, leaving dangling portal
  region ids that later index `visited_epoch_[0xFFFFFFFF]` (OOB write).
  `is_region_graph_fresh` makes the same size-only check while its comment
  claims shape detection.

## Medium — diagnostics (S4)

- **M9.** `bench/tess_diagnostics_alloc_hooks.cc:123-137` —
  `release_bytes` records a deallocation before the null check, but
  `operator delete(nullptr)` is a legal no-op → spurious
  deallocations > allocations imbalance for leak-balance consumers.
- **M10.** `bench/tess_diagnostics_alloc_hooks.cc:18-24` — the sanitizer
  branch keys on `__SANITIZE_ADDRESS__`, which MSVC `/fsanitize=address`
  also defines, then includes `pthread.h` → Windows+ASan compile break
  (and MSVC ASan doesn't call `__sanitizer_*` hooks anyway).
- **M11.** `include/tess/diagnostics/export.h:42-57` —
  `capture_timing`/`capture_diagnostics` read trace/counter state with zero
  synchronization and no documented same-thread requirement; the natural
  ImGui usage (sim thread traces, UI thread snapshots per frame) is a data
  race. Document the same-thread contract at minimum.

## Medium — tests / bench / CI / process

- **M12.** `bench/tess_parallel_bench.cc` — all 10 `parallel/*` benchmarks
  are compiled into `tess_bench` but executed nowhere in CI: no thresholds
  file, no smoke `add_test`, excluded from baselines. Crash or regression is
  invisible; nothing documents this as deliberate. (Verified by name-diffing
  all 125 registered benches against all 115 threshold entries — every other
  family maps 1:1.)
- **M13.** `tools/benchmark_trends.py:134` — reads only
  `block/storage/key.json` from a 7-file baseline artifact; `--benchmark
  path/...` silently renders "n/a". The 415-line trends tool (and the
  metadata tool CI runs) have zero tests.
- **M14.** `tests/tess_phase_executor_test.cc:150-172` — warm-dispatch
  no-alloc test counts allocations process-globally across 2 live worker
  threads; any runtime/stdlib allocation on a worker (platform-dependent)
  fails it. Latent flake.
- **M15.** `tests/imgui_stub/imgui.h:13` — stub `Text` lacks
  `IM_FMTARGS(2)`, so `-Wformat` never checks `panels.h` format strings; a
  mismatch would only surface in a real-ImGui `-Werror=format` consumer.
  One-line fix.
- **M16.** Doc drift from the 2026-07-07 gate promotions: `README.md:56,65-66`
  and `tools/check_public_surface.py:2` and `tests/AGENTS.md:350` still call
  the Windows MSVC job and public-surface check "advisory"; both are required
  gates (`ci.yml:121-125,277-282`). `README.md:71-73` omits the new
  `topology`/`diagnostics` threshold suites and the advisory GCC job.
  `ci.yml:274-280` carries two contradictory comment blocks on the Windows
  job.
- **M17.** `tools/git_hooks.py:40-59` — `PRIVATE_PATTERNS` has no pattern for
  the private sibling-project name, so the policy whose violation forced the
  2026-07-09 history rewrite has zero automated enforcement (pre-commit or
  CI backstop). The file's split-string pattern style exists precisely to
  make adding a self-matching pattern safe.

## Low (hardening, pathological inputs, gaps)

- `phase_executor.h:151-196` — thread creation not exception-safe (partial
  pool construction unwinds a vector of joinable threads → `terminate`);
  throwing callback under `ScopedThreadPhaseExecutor` → `terminate`,
  undocumented there.
- `sim/path_agent.h:138,228` + `path_agent_tick.h:71-74` — blocked-retry
  budget double-counts per cycle yet resets on every `Found` re-path, so an
  agent blocked by a permanently parked agent re-paths every tick forever and
  forces full resubmission of all goal-holders each tick (per-tick pathing
  work never quiesces). Intent ambiguous; document or age the budget.
  **Resolved in v0.12:** occupancy/reservation waits now retain their route
  without submission and exhaust one bounded consecutive-block budget.
- `sim/time.h:68-75` — `available` division can round up across an integer
  boundary → one tick borrowed early, self-correcting (benign; noted for the
  record).
- `path.h:1333-1336` / `detail/weighted_batch.h:190-193` — two-call
  distance-field build/read validates residency only (compile-time true for
  dense), not content/world-identity/Tag; dense world-swap or edit between
  calls silently follows stale gradients. Misuse-dependent; document the
  contract or fingerprint the field.
- `path.h:1139-1141` — rebuild-from-own-product self-aliasing
  (`build_weighted_portal_route_product(..., product.waypoints(), ...,
  product)`) clears the buffer the span points into → UB. Plausible call
  shape; take by value or document.
- `route_cache.h:180-183` vs `portal_segment_cache.h:93` — cap value 0 means
  "unlimited" in one cache and "store nothing" in its sibling; a user
  disabling route caching with 0 gets unbounded growth.
- `path_runtime.h:115-156` — `PathTicket.generation` (uint32) wraps after
  2^32 `clear_requests()`; retained tickets alias (~49 days at 1 kHz).
  Only unguarded generation counter in the subsystem.
- `detail/astar.h:237,344` — plane-gap heuristic sums with unsaturated
  uint32 `+` (sibling code uses `saturating_add`); unreachable with dense
  node arrays today.
- `core/shape.h:106-116,227-234` — compile-time UInt128 triple products wrap
  mod 2^128, silently defeating the "fits uint64" static_asserts for
  pathological shapes (`local_tile_count == 0` → OOB where a compile error
  was intended); no static_assert that axes fit int64.
- `block/block.h:329-334` — `ChunkView::is_boundary` returns false for
  1-tile-wide chunk axes even when neighbor chunks exist along that axis;
  semantics undocumented.
- `storage/chunk_meta.h:48-51` (also `topology.h:594`) — `box_axis_end`
  int64 cast flips negative for extents ≥ 2^63 → corrupted dirty bounds.
  Pathological input.
- `topology.h:928-937,1142-1157` — sparse staleness uses per-world
  generation clocks; a graph built on world A can validate fresh against a
  same-shape world B with identical load order. The library documents this
  collision class for `residency_fingerprint` but the topology checks don't
  consume it.
- `tests/tess_queued_test.cc:1401,1482` — uncapped spin-wait barriers with
  no ctest TIMEOUT → a concurrency regression hangs ~1500 s instead of
  failing fast.
- Untested surfaces (grep-verified): `PrecheckStatus::InvalidStart`;
  `MovementFailureCounts.reserved/stale_version/stale_topology`; four
  `PathCounters` fields; `TESS_ASSERT_MSG` (also unused);
  UInt128 negative-ctor assert.
- `tests/tess_smoke.cc:5-11` — version check is tautological (compares
  macros to themselves).
- `tests/tess_path_agent_test.cc:410,459`, `path_agent_tick_test.cc:413` —
  warm no-alloc tests discard the result and assert only `count()==0`; an
  early-return no-op also passes (stronger sibling pattern exists at :432).
- `tests/allocation_counter.cc:38-63` — relaxed ordering can *undercount*
  cross-thread allocations near scope boundaries (weakens `==0` guards,
  never false-fails).
- `bench/thresholds/path.json` — two `manual_time` notes cite a "1 ms
  ceiling" while values are 1500/910 ns; threshold gates are single-shot
  (acceptable while headroom stays ~5x).
- Infra nits: steamdeck base image tag unpinned (acknowledged in-file);
  CI clang-tidy/cppcheck unpinned from apt while `docs/dependencies.md:98`
  documents cppcheck 2.20.0 (Ubuntu 24.04 ships 2.13.x); `check_limit`
  silently ignores typo'd limit keys; `deck:45` help text omits two env vars
  it honors; `.gitignore` lacks `.pytest_cache/`; `docs/dependencies.md`
  omits `actions/cache@v4` from the actions inventory; the GCC job's
  "flip to required after two green runs" promotion (2026-07-07) may be
  overdue.

## Verified clean (positive assurance)

- A* cores: unit-cost two-bucket open list, admissible/consistent
  heuristics, all fast-path early-outs proved sound and optimal.
- Incremental vs full topology: provably convergent for the documented
  contract (correct dirty set, same world/shape).
- Sparse residency: ChunkDirectory probing/backward-shift erase, LRU and
  slot bookkeeping, within-world staleness — all hand-traced clean;
  `Indeterminate` reachability logic sound.
- `uint128.h` runtime arithmetic; TraceBuffer/warning-sink ring math; trace
  macro hygiene (single evaluation, true zero-overhead when disabled);
  ImGui panel gating; diagnostics CMake wiring.
- Planner hazard/phase logic in `queued.h`; executor protocol under the
  single-caller contract (memory ordering verified); task lifetimes.
- CI: actions SHA-pinned, least-privilege permissions, no secrets, no
  swallowed failures beyond documented advisory jobs; public-header manifest
  in exact sync; no tracked private-name hits; no committed artifacts;
  compiled development-requirements pins consistent.
- Test suite quality: oracle-backed path tests, fixed seeds, deterministic
  iteration, real negative-path/eviction coverage, no vacuous or tautological
  tests beyond the smoke-version nit.
- Live run: 517/517 tests pass under ASan/UBSan (dev-asan preset, 27.9 s).

## Suggested remediation order

1. **H1+H2** (one design decision: how failure products/fields invalidate —
   simplest is a world content-fingerprint dependency or `is_valid == false`
   for empty dep sets) with regression tests for unreached-chunk edits and
   failure-product replay.
2. **H3+M2** — document the executor dispatch contract, add debug guards.
3. **M1** (set pathing-dirty on partial execution), **M7/M8** (cheap range
   and shape guards in topology), **M3/M4** (route-cache API), **M5/M6**
   (document heuristic tier + world binding).
4. Diagnostics trio **M9-M11** (all small).
5. CI/test batch **M12-M17** (parallel bench gate, trends tool, doc drift,
   sibling-name hook pattern).
6. Low-severity hardening opportunistically alongside touched files.
